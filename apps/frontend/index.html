1. Keycloak Get a token for one of the seeded demo users.
2. Gateway Validate JWT, tenant, audience, and roles.
3. Orchestrator Choose a workflow and publish tool requests.
4. Kafka Carry requested and completed tool events.
5. Workers Run SQL or report work against the sample services.

Tutorial

Use these steps after the Compose stack is running.

  1. Start the stack with docker compose up --build.
  2. Select a seeded user and press Login.
  3. Pick an agent and sample message.
  4. Press Run agent and inspect the JSON response.
  5. Watch worker activity with docker compose logs -f orchestrator sql-worker report-worker db-access.

Seeded Users

Each user proves a different role path.

User Best test
world-analyst World DB SQL and report succeed; procurement access is blocked.
procurement-analyst Procurement DB SQL succeeds; World DB is denied.
source-auditor Both agents are callable, but only World DB is readable.
data-admin Both agents and both database permissions are allowed.
